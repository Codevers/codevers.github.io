Css选择器篇

发表于 2014-02-02 | 更新于 2019-03-26 | 分类于 Css学习 | 评论数： | 阅读次数：

根据w3c标准，一个网站的代码层面分为结构，表现，行为，结构与表现的分离，是w3c所提倡的，不在像以前table布局时代，结构方面的代码十分臃肿，样式和结构的混杂，但现在xhtml+css布局时代的到来，彻底的回归了html、css的本质工作，代码更为高效漂亮，其中颇为重要的概念，Css选择器，这是结构与表现分离最基本的形式，也是找寻样式的必要手段，下面总结一下常用的css选择器。

一般选择器分为（普通选择器）（高级选择器）：
普通选择器（ie6所支持的选择器，最常用）：

1: 元素选择器：最最常用的选择器，每一个网页都有用到，比如reset需要用到，给页面重置html的css浏览器默认属性那是相当有必要的，基本的形式为：h1{color:#f00;}

这里的意思是所有的h1标题的颜色都为红色，覆盖掉浏览器和用户代理的默认样式，默认是黑色的，选择器都有权重，这里的元素选择器的权重为1，除了浏览器默认外它的权重是最低的，所以会覆盖掉默认样式。

2：类选择器：也是最常用的选择器之一，以“.”符号代表一个类选择，它允许一个页面出现多次，可重复使用，基本形式为：.demo{color:#f00;}，引用方式为：&lt;div class="demo"&gt;&lt;/div&gt;，它的权重比关系比元素选择器大些，为10的权重，由于可重复使用，可运用组合的方式来拼接特定样式，适当的运用组合的方式来表现样式，可大大增加css样式的灵活性。

3：id选择器：以“#”符号代表一个id选择，在一个页面里只允许出现一次，具有唯一性，不能被重复使用，（注意是一个命名只允许出现一次，不能有重复命名，当然同样的命名二次引用也有效，不过不符合标准）基本形式为：#demo{color:#f00;}，引用方式为：&lt;div id="demo"&gt;&lt;/div&gt;，由于具有唯一性，一般用于模块式的css命名，比如(#nav,#header,#wrapper)等，切记一定要慎重的使用id还是class，因为有时候确定了是单独的模块，但是做好后，又要求这里也要一个一样的，好吧回头改变选择器，还不能确定各个元素之间的关系，所以要慎重，id适用于大的模块，他的权重较高，为100权重比。

4：行内样式：这种算不上是选择器，因为并没有体现出结构与表现的分离，它直接做用于html代码中，基本形式为：&lt;div style="color:#f00;"&gt;&lt;/div&gt;，这种方式的css样式权重高，为1000的权重比关系，当然了，要说权重最高应该是!improtant了，优先读取，但是只限于developer了，最终权重关系取决于用户，比如说用户调大了字体大小，满足用户需要选择方式，才是好的设计，

5：后代选择器：一种结合各类选择器和元素选择器的选择，基本形式为：.demo h1{color:#f00;}，class为demo的元素里的所有h1标题颜色都为红色，直接作用于所有后代，包括子孙关系的h1，必须是对应的祖先元素的class为demo，可喜的是，它们两的结合方式使得权重大了一个，为11权重比关系，这也是常用的选择方式，比如也可以这样：(#nav li{},span i{} )等方式。

6：通配选择器：直接翻牌所有的浏览器默认样式，只要一个符号，即可实现覆盖所有所自定义的样式，基本形式为：`{color:#f00;}`，所有的元素的字体样色都为红色，不分祖先元素子元素什么的，一般用于重置所有的样式，不过我不建议使用，很耗资源，如果是transitional的文档类型，那么连带着过时的标签都渲染了，比如( <b> <i> <font> )等.

下面是高级选择器（也就是ie6所不支持的选择器，所以不常用）

1：属性选择器：根据元素的属性和属性值来进行选择；如：”img[alt]“意思是在结构里有img，并且有alt的属性，应用设置的样式

2：子选择器：形式跟后代选择器相似，以”>“符号选择，后代选择器是选择一个元素的所有后代，而子选择器只选择元素的直接后代，他是以父级的以下的直系子元素，如果有其他元素包括了a元素，那么就是孙子元素了，这样对孙子元素无效，基本样式为：#content&gt;a{color:#f00;} html:&lt;div id="content"&gt; &lt;a href=""&gt;web&lt;/a&gt; &lt;p&gt;&lt;a href=""&gt;web2&lt;/a&gt;&lt;/p&gt; &lt;/div&gt;;那么第一个a元素web有效，而p内的a元素web2无效。当然了ie6是不支持的，而ie7虽然支持，但在父与子元素之间添加注释会有问题。 《技巧：利用通配符可以实现ie6不支持的子选择的效果》
栗子：#container a{text-decoration:none;} #container a*{text-decoration:underline;}那么除了container a内的元素外，其余被覆盖,无下划线。

3：相邻同胞选择器：前提是有共同的父元素，可以使在一个父元素下，同胞选择器会作用于一个元素之后的元素，基本形式为：p + p{color:#f00;} ,&lt;div&gt;&lt;p&gt;web01&lt;/p&gt;&lt;p&gt;web02&lt;/p&gt;&lt;/div&gt;,在同一父级下，那么只对web02起作用.字体红色 ，（与子选择相同，ie7中添加注释会有问题 ）

最后总结：常用选择器基本上就是这些，都是较常用的，所以这里也适当的汇总一下，当然了还有不少选择器，那算属于较偏的了，比如span .div{color:#f00;}，这种就代表只限于span内的class为div的样色、
