Css定位篇

发表于 2014-01-07 | 更新于 2019-03-26 | 分类于 Css学习 | 阅读次数：

Css定位属性，顾名思义，对元素进行一个定位操作，并且相应的去移动它，就像你现在用手指按住一枚硬币，对，硬币会乖乖按照你想移动的方向而改变了原本位置。那么问题来了，在Css中用什么来实现定位？用什么来移动元素呢？移动过程中有哪些规则？

定位有很多种，每一种的规则都不同，但目的都是相同的，移动元素，那么移动的过程中需要注意哪些呢？以下是博主近期对定位属性的浅薄理解：

如果你任真看完本章后，你就会搞明白布局中最重要的概念之一（定位），一：文档流关系，二：z-index层叠关系，三：脱离文档流状态。搞透彻了证明你布局能力跨了一大步。^-^

注： 在说明之前十分有必要说一下文档流的问题：

什么是文档流？文档就是文档，没有什么，“流”代表什么？小溪流水、瀑布流水等， 都是所谓的“流”。呵呵，抛砖引玉，那么在html中文档流类型有两种：

普通文档流：一般是指html代码按照顺序显示，即先写的标签先显示，后写到标签后显示，像瀑布的水一样从上而下流下，这是普通文档流。

特殊文档流：一般是指脱离了文档流，不在按照普通文档流的顺序显示了，脱离文档流元素不再受普通文档中的普通布局元素影响，是一个单独跳出来的一个元素。

一个页面，不光有表面上的x轴、y轴，其实页面是有z轴的，它的z轴就在z-index这一层，在我们制作的页面，上面所有的普通元素都处在同一层上，即Z轴：z-index:0这一层，但是元素可以通过更改页面层级，从而做到能轻易覆盖其他的层，如float，relative，absolute，设置这些可以让元素”浮”起来，就是说z-index值大于z-index：0上，大概是z-index:1，可以覆盖其他一切小于z-index：1的元素，注意需要定位才可激活z-index，才能在定位元素上使用，但float情况特殊，并没有”浮”起来，层叠位置任然处在z-index：0上，理解这些下面就好明白了。
（首先说明：设置了position：relative/absolute/fixed会激活 left/right/top/bottom以及z-index，默认情况下未激活，设置了也无用！）

静态定位

在普通文档流中，所有的元素都是静态定位，它的定位方式是以在html代码书写中的先后顺序进行排列的，层叠顺序按照代码的先后顺序来的。

相对定位

首先它并没有脱离了文档流，但是它”浮”了起来！就是说它的值大于z-index:0，重要的是它保留了自己的占位信息！其他普通元素看它是存在的，不会去无视它，他的移动方式是以自身为基准的，也就是说以自己为起点，而且他的移动却总是可以覆盖其他普通元素，这都是因为它”符”了起来，z-index已经大于其他的元素了。为z-index：1，之前我说过，对普通元素设置z-inex是无效的，因为普通元素未激活z-index，那么可以通过给定位元素设置z-index可以改变他们的层叠关系，只有对定位元素设置z-index才有效。

绝对定位

脱离了文档流，而且也是”浮”了起来，也是z-index大于0，可惜它并没有保留自己的占位信息!所以它对于其他普通元素眼里是不存在的，所以普通元素会忽略它进行覆盖，！但是absolute的值大于z-index:0，所以最终结果是absolute覆盖普通元素，可以通过对absolute元素设置z-index值改变层叠关系，如z-index:-1；这样普通元素就能覆盖绝对定位元素了，

那么它的移动方式是相对于父级或者祖先元素设置了定位元素为起点，注意：是任何定位属性，不是只有relative，如果没有，那么它就会相对于浏览器窗口，相对当前浏览器大小，注意并不是body，不信可以做个demo，最后一个有趣的现象是，设置absolute可以改变元素的display类型为inline-block，可以设置height、width。

固定定位

首先从字面上意思就看出来了，顾名思义，固定，把一个元素固定住，怎么滚动移动都不行，当然了，默认定位方式是scroll滚动，注意它的定位方式始终以浏览器窗口为基准，只是自己固定住了，并且固定定位脱离了文档流，不在保留占位信息，z-index为1，所以他也可以覆盖普通元素。注意这种属性在ie低版本里不支持。

浮动定位

脱离了文档流，说法不一，有人说进入了特殊的流，而且既然脱离文档流，文本为什么不会无视，而absolute会无视，这样就很奇怪，例子：文本环绕，不过我认为是脱离了文档流会更加理解，但是它不会让元素”浮”起来，也就是还在z-index:0，它的定位方式是以left/right为进行移动的，直到碰到包含框为止，由于是脱离了文档流，那么如果有非浮动元素与它发生重叠，会被浮动元素覆盖。

如果一个自适应高度的父元素的子元素全为float元素的话，那么高度就不会自适应，我想是因为浮动脱离文档流，父级不认识子级，解决方法：清除浮动就可以了，很奇怪，有趣的是float也可以改变元素的display类型为inline-block，而且由于它是脱离了文档流所以会覆盖普通元素，所以就算对float、absolute设置了其它的display类型都是无效的，（这样的话就解释了ie6的浮动双倍边距bug的情况）
