ViewPort 视区概念

开发手机页面时往往会遇到手机的分辨率或屏幕大小不同的问题，如何使我们编写出来的页面大小能适合各种手机呢？html5中的 viewport 能帮你做到！

其实手机浏览器是把页面放在一个 虚拟的“窗口”(viewport) 中，通常这个虚拟的“窗口”(viewport)比屏幕宽，这样就不用把每个网页挤到很小的窗口中(这样会破坏没有针对手机浏览器优化的网页的布局)，用户可以通过平移和缩放来看网页的不同部分，让网页开发者来控制 viewport 的大小和缩放，其他手机浏览器也基本支持。

正因为虚拟窗口比设备尺寸大，所以需要相应设置，达到虚拟窗口==设备尺寸，这是很重要的一步。

该元信息表示的意思为：

1.设置屏幕宽度为设备宽度 width=device-width

2.设置页面最小缩放比例为1，等同于禁止了缩放 initial-scale=1

3.设置页面最大缩放比例为1，等同于禁止了缩放 maximum-scale=1

maximum-scale都将被忽略，因为根本不可能缩放。

参数

  • width：控制 viewport 的大小，这个很关键，可以指定的一个值，如果 600，或者特殊的值，如 device-width 为设备的宽度(单位为缩放为 100% 时的 CSS 的像素)。

  • height：和 width 相对应，指定高度

  • initial-scale：初始缩放比例，也即是当页面第一次 load 的时候缩放比例。这是一个浮点值，是页面大小的一个乘数。例如，如果你设置初始缩放为1.0，那么，web页面在展现的时候就会以target density分辨率的1:1来展现。如果你设置为2.0，那么这个页面就会放大为2倍。

  • maximum-scale：允许用户缩放到的最大比例。这也是一个浮点值，用以指出页面大小与屏幕大小相比的最大乘数。例如，如果你将这个值设置为2.0，那么这个页面与target size相比，最多能放大2倍。

  • minimum-scale：允许用户缩放到的最小比例。

  • user-scalable：用户是否可以手动缩放，如果设置为yes则是允许用户对其进行改变，反之为no。默认值是yes。

所有的缩放值都必须在0.01–10的范围之内。

最后总结

  • 必须要设置width=device-width参数，以用来确定可视窗口是否等于设备width，因为页面是在视区“虚拟窗口上”展示的，而默认视区宽度比设备宽度大，所以要：视区宽度=设备宽度，通俗理解为：把页面展示在屏幕上，等同于在屏幕上布局页面

  • 通常设置页面不允许缩放，随意缩放有可能会破坏网页结构，造成错乱

  • 最大缩放比例 和 最新缩放比例 要设置为1.0，即原始缩放比例
